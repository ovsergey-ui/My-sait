# Портфолио: продукты с ИИ, интерфейсы и цифровые системы

Статический одностраничный сайт-портфолио в стиле premium dark UI по техническому заданию из `TZ.md`. Язык интерфейса и контента: **русский** (профессиональный тон).

## Стек и подход

| Слой | Файл / технология |
|------|-------------------|
| Разметка | `index.html` |
| Стили | `styles.css` |
| Шрифты | Google Fonts: **Onest** (текст), **Unbounded** (заголовки); направление визуала — по `SKILL.md` (frontend-design): отказ от «шаблонной» пары Inter/Space Grotesk, акцентная палитра, асимметрия hero, фоновая глубина |
| Скрипт | Встроенный в конец `index.html` (без отдельного `.js`): плавный скролл по якорям, появление блоков при скролле, лёгкое свечение за курсором, spotlight в секции «Преимущества», год в подвале |

Исходное ТЗ предполагало React / Tailwind / Framer Motion; **MVP реализован на чистом HTML/CSS** с минимальным JavaScript для интерактива.

## Секции страницы

1. **Первый экран (hero)** — заголовок, CTA и портрет (`Foto/My1.png`).
2. **Обо мне** — позиционирование, список направлений, бейджи и макет-иллюстрация.
3. **Услуги** — сетка из 6 карточек.
4. **Процесс** — 4 этапа; на широких экранах анимация линии прогресса при появлении в зоне видимости.
5. **Проекты** — 4 карточки: у первого кейса превью-картинка (`projects/ai-mechanic-assistant.png`), у остальных — градиентные заглушки; оверлей при наведении.
6. **Преимущества** — текст и spotlight по курсору внутри секции.
7. **Технологии** — бейджи инструментов (названия продуктов на латинице).
8. **CTA** — призыв и кнопки.
9. **Подвал** — контакты (ссылки-заглушки: заменить на реальные).

## Как открыть

- Откройте `index.html` в браузере двойным щелчком, или  
- Поднимите локальный сервер в папке проекта (например, `npx serve`), если нужны те же условия, что и при разработке.

## Что настроить под себя

- Логотип в шапке: сейчас `студия.систем`.
- Почта и соцсети в подвале (`hello@example.com`, `t.me`, `github.com` и т.д.).
